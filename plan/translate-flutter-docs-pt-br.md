## Instruções de Uso do Agente de Tradução

Para cada etapa, siga este fluxo:

### 1. Listar arquivos da etapa
```bash
find src/content/[diretório] -name "*.md" | sort
```

### 2. Para cada arquivo, usar o agente translate-pt-br
O agente está em `.claude/agents/translate-pt-br.md` e inclui:
- Regras de tradução
- Preservação de links (CRÍTICO!)
- Validações obrigatórias
- Formato de commit

### 3. Validação obrigatória (BLOCKING)
Antes de cada commit, executar:

```bash
# a. Verificar reference keys vazias (DEVE retornar vazio)
grep -E "\[.*\]\[\]" arquivo.md

# b. Verificar todas as referências têm definições
grep -oE "\]\[([^\]]+)\]" arquivo.md | sed 's/\]\[//' | sed 's/\]//' | sort -u > /tmp/refs_used.txt
grep -oE "^\[([^\]]+)\]:" arquivo.md | sed 's/\[//' | sed 's/\]://' | sort -u > /tmp/refs_defined.txt
comm -23 /tmp/refs_used.txt /tmp/refs_defined.txt
# DEVE retornar vazio!

# c. Verificar reference keys em inglês
grep -E "^\[[^]]*[áéíóúãõçÁÉÍÓÚÃÕÇ][^]]*\]:" arquivo.md
# DEVE retornar vazio!

# d. Verificar âncoras em inglês
grep -E "\{:#[^}]*[áéíóúãõçÁÉÍÓÚÃÕÇ][^}]*\}" arquivo.md
# DEVE retornar vazio!

# e. Verificar metadata
grep "ia-translate: true" arquivo.md
# DEVE retornar a linha!
```

### 4. Commit individual com co-author
```bash
git add caminho/para/arquivo.md
git commit -m "$(cat <<'EOF'
translate: caminho/para/arquivo.md

- Add ia-translate: true metadata
- Translate title and description to PT-BR
- Translate content while keeping technical terms in English
- Preserve all links, code blocks, and formatting

Co-authored-by: Ulisses, Mago do Flutter <ulisseshen@gmail.com>
EOF
)"
```

### 5. Push regular
```bash
# A cada 10-20 arquivos traduzidos:
git push -u origin claude/plan-flutter-docs-pt-br-011CUwUz8aJnpAeBJvsH6nDd
```
